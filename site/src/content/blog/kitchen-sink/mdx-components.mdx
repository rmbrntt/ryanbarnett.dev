## Component Imports

MDX allows you to import and use any Astro or React component:

```mdx
import Callout from '@/components/Callout.astro'
import { Icon } from 'astro-icon/components'
import CustomComponent from '@/components/CustomComponent.astro'
```

## Advanced Patterns

### Component with Props

While you can't pass props directly in MDX, you can create variations:

<div class="space-y-2">
  {['primary', 'secondary', 'destructive'].map(variant => (
    <button 
      class={`px-4 py-2 rounded font-medium ${
        variant === 'primary' ? 'bg-primary text-primary-foreground' :
        variant === 'secondary' ? 'bg-secondary text-secondary-foreground' :
        'bg-destructive text-destructive-foreground'
      }`}
    >
      {variant.charAt(0).toUpperCase() + variant.slice(1)} Button
    </button>
  ))}
</div>

### Dynamic Lists

<div class="divide-y">

## Best Practices

<Callout variant="tip" title="MDX Component Tips">
  1. **Import once** - Import components at the top of your MDX file
  2. **Use semantic HTML** - Maintain accessibility with proper HTML elements
  3. **Responsive by default** - Always consider mobile layouts
  4. **Dark mode support** - Use Tailwind's dark: variants
  5. **Performance** - Avoid heavy JavaScript in MDX components
</Callout>

This guide demonstrates the flexibility and power of MDX components in your blog posts. Combine these patterns to create rich, interactive content!
